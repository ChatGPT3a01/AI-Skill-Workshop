Part 1 · AI Skill 概念速覽

什麼是 Agent Skills？

理解 AI Skill 的核心概念，從「會用 AI」升級到「讓 AI 照你的方式做事」

Agent Skills 流程變現 三大平台

🍳 Skills = AI 的食譜

你有沒有遇過這個情境：花了 10 分鐘教 AI 怎麼做一件事，結果下次開新對話，又要重新教一遍？

這就好像每次做菜都要重新發明食譜——明明上次已經煮得很好了，為什麼不把步驟寫下來呢？Agent Skills 就是你寫給 AI 的「食譜」，讓 AI 每次都能按照你設定的流程，穩定地完成任務。

🧑‍🍳 食譜 vs Skill 對照表

做菜的食譜對應到 Skill說明
食譜名稱Skill 名稱例如「Email 回覆助手」「會議記錄整理」
食材清單輸入參數AI 需要你提供什麼資料才能開始工作
烹飪步驟執行規則AI 要依序完成的每一個動作
成品照片輸出格式最終交付的結果長什麼樣子
注意事項防呆機制什麼情況該拒絕、什麼錯誤要避免
💡 重點：Skill 就是一個 Markdown 檔案（SKILL.md），不需要寫程式！只要你會打字、會列清單，就能建立自己的 AI Skill。從此告別「每次重新教 AI」的窘境。
⚠️ 常見誤解：很多人以為 Skill 就是 Prompt——錯！Prompt 是一次性的指令，用完就丟；Skill 是結構化的檔案，存在硬碟裡，可以重複使用、分享給團隊、版本控制。本質上是兩種完全不同的東西。

⚡ Skills vs 一般 Prompt

Skill 和一般 Prompt 看起來都是「告訴 AI 該怎麼做」，但本質上有六個關鍵差異。理解這些差異，你才能真正體會為什麼要花時間學 Skill：

比較項目一般 PromptAgent Skill
保存方式存在聊天紀錄裡，容易遺失獨立的 .md 檔案，永久保存
觸發方式每次手動貼上或重新打字AI 自動偵測相關情境，一鍵載入
可維護性改了這次，下次又忘記修改檔案即可，所有對話自動生效
共享性只能截圖或複製文字分享整個資料夾複製即可分享給團隊
載入效率每次都消耗完整 Token按需載入，節省 Token 用量
擴展性Prompt 越長越難維護可拆分多檔案、引用外部資源
🎯 一句話總結：Prompt 是「用完即丟的便利貼」，Skill 是「寫好永久用的操作手冊」。當你發現自己重複貼同一段 Prompt 超過 3 次，就是該把它升級成 Skill 的時候了。

✨ Skill vs Prompt（Before/After 對比）

讓我們用一個實際的工作場景來感受差異。想像你是一位客服主管，團隊有 5 個人，每個人回覆客戶信件的風格都不一樣，品質也參差不齊。導入 Agent Skills 前後，工作方式會有什麼根本性的改變？

❌ Before：沒有 Skills

  • 每次重新教 AI：開新對話就要重新交代一次規則，浪費 5~10 分鐘
  • 品質不穩定：同一個任務，每次給出的結果格式、語氣都不一樣
  • 無法分享給同事：你的「秘密 Prompt」只存在你的聊天紀錄裡
  • 離職知識就消失：專業知識隨著人員異動而流失
  • 難以迭代改進：不知道上次用的是哪個版本的 Prompt

✅ After：有了 Skills

  • 一鍵觸發：AI 自動偵測情境，載入對應 Skill，零秒啟動
  • 品質穩定一致：每次都按照相同的步驟和格式執行
  • 整個資料夾分享：把 skills/ 資料夾傳給同事就搞定
  • 知識永久保存在檔案中：人員異動不影響，Skill 就是組織的知識庫
  • 版本控制容易：用 Git 追蹤每次修改，隨時可以回溯
🎯 核心價值：Skills 讓你的專業知識從「存在腦中」變成「存在檔案中」，從「個人經驗」變成「團隊資產」。這就是「專業流程變現」的第一步。

📋 拆解你的工作流程

把你腦中的專業知識，拆成 AI 看得懂的步驟——這是建立 Skill 最重要的前置作業，也是整個課程的核心技能。很多人以為「我的工作太複雜，AI 做不來」，其實不是做不來，而是你還沒把流程拆解得夠清楚。

📐 拆解三原則

原則說明正確範例錯誤範例
足夠具體 每一步都可以明確執行，不需要額外猜測 ✅「分析來信的情緒傾向（正面/中性/負面）」 ❌「處理一下這封信件」
有順序性 前後步驟有邏輯關聯，不能跳著做 ✅ 先分析情緒 → 再組織回覆 → 最後檢查語氣 ❌ 隨便從哪一步開始都行
可判斷完成 每個步驟有明確的產出物，知道做完了沒 ✅「輸出：情緒判斷結果（正面/中性/負面）」 ❌「大概看一下就好」

📝 實際演練：Email 回覆流程拆解

以「收到客戶來信後的回覆流程」為例，看看怎麼把日常工作拆解成 AI 可執行的步驟：

  1. 辨識信件類型：是詢問、投訴、感謝，還是合作邀約？
  2. 分析來信情緒：對方的語氣是友善、中性還是不滿？
  3. 提取關鍵資訊：對方的核心需求是什麼？有沒有時間限制？
  4. 決定回覆策略：根據信件類型和情緒，選擇適合的回覆框架
  5. 撰寫回覆內容：按照策略撰寫，確保語氣一致
  6. 自我檢查：確認沒有遺漏重點、語氣是否恰當
💡 訣竅：試著想像你在教一個「很聰明但完全不懂你工作」的新人。你會怎麼一步步教他？那些步驟，就是你的 Skill 內容。先求有，再求好——Skill 的好處就是可以持續迭代。

🔄 從流程到 Skill

上一張投影片我們學會了如何拆解工作流程，現在來看看這些拆解出來的步驟，如何對應到 SKILL.md 的各個段落。這是從「腦中知識」到「可執行檔案」的轉換過程。

SKILL.md 的結構其實就是一份「超級詳細的工作說明書」，每個段落都有明確的用途：

你的流程步驟對應 SKILL.md 段落具體內容
這個任務要做什麼？ ## 用途 清楚描述這個 Skill 解決什麼問題、適用什麼場景
開始前要確認什麼？ ## 前置檢查 使用者需要提供什麼資料？缺少什麼要先問？
具體怎麼做？ ## 執行步驟 AI 要依序完成的每一個動作，越具體越好
做完要長什麼樣子？ ## 輸出格式 最終結果的格式、結構、長度等規範
什麼事情絕對不能做？ ## 安全規則 防呆機制、禁止行為、邊界條件處理
做一次示範 ## 範例 給 AI 看一個完整的輸入→輸出範例

🎯 用途要精準

不要寫「幫我處理 Email」，要寫「根據收到的客戶來信，分析情緒和類型，生成符合公司語氣的專業回覆」。越精準，AI 越容易判斷何時該啟用這個 Skill。

📋 步驟要有序

用數字編號列出每一步，而且每一步都要有明確的輸出。例如「步驟一：判斷信件類型 → 輸出：類型標籤（詢問/投訴/感謝/合作）」。

🛡️ 安全不能少

告訴 AI 什麼情況要拒絕（例如：「如果來信涉及法律糾紛，回覆『建議諮詢法律顧問』」），避免 AI 在不該回答的地方亂回答。

🖥️ 三大 AI 平台總覽

目前支援 Agent Skills 的主流平台有三個，各有不同的適用場景。選擇哪個平台取決於你的需求、預算和技術背景。這門課會帶你在這三個平台上都實際操作。

🌐 Antigravity（Google）

免費使用，提供圖形化網頁 IDE 介面。Skill 放在 .agent/skills/ 路徑。

不需要安裝任何軟體，打開瀏覽器就能用，最適合入門學習。視覺化操作，拖曳上傳 Skill 即可執行。

適合：非技術背景、初次接觸 AI Skill 的學員

⚡ Claude Code（Anthropic）

專業開發者工具，終端機 CLI 介面。Skill 放在 .claude/skills/ 路徑。

內建 16+ 官方 Skills，程式碼能力強，支援 /skill 指令直接呼叫，是進階使用者的首選。

適合：工程師、需要寫程式的進階使用者

🔷 Gemini CLI（Google）

免費使用，命令列工具。自動讀取工作區檔案中的 Skill。

完美整合 Google 生態系（Drive、Docs、Sheets 等），適合已在使用 Google 服務的團隊。

適合：Google Workspace 使用者、跨平台協作

項目AntigravityClaude CodeGemini CLI
費用免費使用付費（需 API Key）免費使用
介面網頁 IDE（圖形化）終端機 CLI（命令列）終端機 CLI（命令列）
Skill 路徑.agent/skills/.claude/skills/讀取工作區檔案
安裝需求不需要安裝npm install -g @anthropic-ai/claude-codenpm install -g @anthropic-ai/gemini-cli
💡 建議學習順序：先從 Antigravity 開始（免費、圖形化、容易上手），熟悉 Skill 概念後，再嘗試 Claude Code 和 Gemini CLI。三個平台的 Skill 寫法幾乎相同，學會一個就能舉一反三。

💻 三大平台操作介面

看完平台總覽後，讓我們實際看看每個平台的操作介面長什麼樣子。了解各平台的介面特色，能幫助你選擇最適合自己的工具。

⚡ Claude Code 終端機介面

Claude Code 是 Anthropic 推出的命令列工具，直接在終端機中與 AI 對話。它的強項在於程式碼生成與檔案操作——可以直接讀寫你的專案檔案，執行終端機指令，非常適合開發者。

npm install -g @anthropic-ai/claude-code

# 在專案目錄中啟動
cd my-project
claude

# 使用 Skill 的方式
# Claude Code 會自動掃描 .claude/skills/ 目錄
# 偵測到相關情境時自動載入對應 Skill

🔷 Gemini CLI 介面

Gemini CLI 是 Google 推出的免費命令列工具，特色是能自動讀取工作目錄中的所有檔案作為上下文。不需要特別設定 Skill 路徑，只要把 SKILL.md 放在工作目錄中，Gemini 就能找到。

# 安裝 Gemini CLI
npm install -g @anthropic-ai/gemini-cli

# 在專案目錄中啟動
cd my-project
gemini

# Gemini 會自動讀取工作區中的 SKILL.md 檔案
# 直接詢問就會觸發對應的 Skill
💡 跨平台小秘訣：三個平台使用的 SKILL.md 格式幾乎相同！你只需要學會寫一次 SKILL.md，就能在三個平台上使用。差別只在 Skill 檔案放置的位置不同。

🏗️ 流程變現五層架構

「流程變現」是這門課的核心理念——把你腦中的專業知識，一步步轉化為可重複執行、可分享、可販售的 AI 技能資產。整個過程分為五個層次：

L1

核心知識

你的專業經驗與 Know-How。例如：十年客服經驗、專案管理方法論、行銷策略思維、財務分析技巧。這些知識目前只存在你的腦中。

L2

流程拆解

將知識轉化為標準作業程序（SOP），步驟化、具體化、可重複執行。例如：收到客戶來信 → 分析情緒 → 選擇策略 → 撰寫回覆。

L3

SKILL.md

將 SOP 寫成 AI 可執行的 Skill 檔案，包含 YAML 前言、執行步驟、安全規則、輸出格式、範例。這是最關鍵的一步。

L4

生成器表單

為 Skill 製作使用者介面，讓任何人（即使不懂 AI）都能透過表單使用你的 Skill。降低使用門檻，擴大受眾範圍。

L5

AI 工作系統

多個 Skill 串接成自動化工作鏈。例如：收信 → 分類 → 回覆 → 歸檔一條龍，打造完整的 AI 工作流。

🎯 這門課帶你走完 L1 → L3，體驗 L4 和 L5！你將親手把自己的專業知識拆解成流程、寫成 SKILL.md、部署到平台上執行，並體驗 Skill 串接的威力。
