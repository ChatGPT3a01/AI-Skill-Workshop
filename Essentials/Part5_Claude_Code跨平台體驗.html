在 Claude Code 使用 Skills

安裝 Claude Code CLI，設定 API Key，用 /skill-name 指令執行 Skill，體驗跨平台部署

💻 Claude Code 🔑 API Key ⌨️ CLI 指令 🌐 跨平台 🧩 內建 Skills

💻 Claude Code 是什麼？

Claude Code 是 Anthropic 官方推出的 AI 開發工具（CLI），讓你直接在終端機中與 Claude AI 互動，並支援完整的 Skills 系統。

  • Anthropic 官方的 AI 開發工具（CLI 命令列介面）
  • 直接在終端機中與 Claude AI 對話
  • 支援 Skills 系統：專案級 + 使用者級
  • 16+ 個內建官方 Skills（pptx、pdf、docx 等）
  • 能讀取、搜尋、編輯專案檔案
  • 支援 MCP 外掛工具，可擴展能力
💡 與 ChatGPT 的差異
ChatGPT 是網頁對話介面；Claude Code 是終端機工具，可以直接存取你電腦上的檔案和專案，更適合實際開發工作。

💳 Claude 定價方案

Claude Code 需要付費使用 API。以下是 Anthropic 的定價說明：

方案 價格 特色
API 按用量計費 依 token 計價 最彈性，用多少付多少
Claude Pro $20/月 包含 Claude Code 使用額度
Claude Max $100/月 更高用量，優先存取
💰 課程建議
初次使用建議先儲值 $5 美元（約台幣 160 元）來體驗。這筆金額可以讓你練習很多次！也可以訂閱 Claude Pro 直接包含額度。

📝 Step 1：註冊 Anthropic Console

要使用 Claude Code，首先需要到 Anthropic Console 註冊帳號並取得 API Key。

註冊步驟

  1. 前往 console.anthropic.com
  2. 點選 Sign Up（註冊）
  3. 填寫 Email 和密碼，完成註冊流程
  4. 驗證 Email 後登入 Console 後台
💡 小提醒
免費註冊，但使用 API 需要先儲值（最低 $5 美元）。支援信用卡或 Debit Card 付款。

🔑 Step 2：建立 API Key

登入 Anthropic Console 後，需要建立一組 API Key 來讓 Claude Code 驗證你的身份。

建立步驟

  1. 登入 Console 後台
  2. 左側選單點選 API Keys
  3. 點選 Create Key（建立金鍵）
  4. 輸入一個容易辨識的名稱（如「my-skill-workshop」）
💡 命名建議
建議取一個容易辨識的名稱，如「my-skill-workshop」或「ai-course-2026」，方便日後管理多組金鍵。

📋 Step 3：複製 API Key

建立完成後，系統會顯示你的 API Key。它的格式是 sk-ant- 開頭的一長串字串。

API Key 格式範例

sk-ant-api03-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx...
⚠️ 極重要警告！
API Key 只會顯示一次！請立即複製保存到安全的地方。如果遺失，只能重新建立一個新的。千萬不要把 API Key 公開分享或上傳到 GitHub！
💡 保存建議
可以存在密碼管理器（如 1Password、Bitwarden）或本地的安全筆記中。絕對不要放在公開的代碼庫裡！

⚙️ Step 4：安裝 Claude Code

使用 npm 全域安裝 Claude Code，只要一行指令就能搞定。

安裝指令

# 安裝 Claude Code（需要先有 Node.js v18+）
npm install -g @anthropic-ai/claude-code

驗證安裝

# 確認安裝成功
claude --version

# 預期輸出：claude v1.x.x
💡 前置需求
需要先安裝 Node.js（建議 v18 以上版本）。如果還沒安裝，可到 nodejs.org 下載安裝。Windows 用戶請選擇 LTS 版本。

🔐 Step 5：使用 API Key 登入

安裝完成後，進入你的專案目錄，啟動 Claude Code 並使用 API Key 登入。

登入步驟

  1. 開啟終端機，進入你的專案目錄
  2. 執行 claude 指令
  3. 選擇「Use an API key」
  4. 貼上你的 sk-ant-... API Key
  5. 出現提示符號 > 表示登入成功！
✅ 登入成功！
看到 > 提示符號就表示可以開始使用了！API Key 會被自動儲存，下次使用不需要重新輸入。

📂 專案級 vs 個人級 Skill

Claude Code 支援兩種 Skill 存放方式，依照使用範圍不同，選擇最適合的層級。

比較項目 專案級（Project） 個人級（User）
路徑 .claude/skills/ ~/.claude/skills/
範圍 只有這個專案能用 所有專案都能用
比喻 🏗️ 工地專用工具箱 🛠️ 你的隨身工具包
適合 團隊共用的 Skill 個人常用的 Skill
Git 版控 ✅ 會被版本控制 ❌ 不會被版本控制

📁 目錄結構

專案目錄/
├── .claude/
│   └── skills/               ← 專案級 Skills
│       └── email-reply-helper/
│           └── SKILL.md
├── CLAUDE.md                 ← 專案設定檔
└── ...

~/.claude/skills/             ← 使用者級 Skills（所有專案共用）
💡 課程建議
今天的課程使用「專案級」，方便團隊分享和版本控制。

🚀 在 Claude Code 使用 Skill

把 SKILL.md 放到 .claude/skills/email-reply-helper/ 後，Claude Code 就會自動偵測並載入你的 Skill。

互動流程

  1. 將 SKILL.md 複製到 .claude/skills/email-reply-helper/
  2. 啟動 Claude Code，輸入客戶來信內容
  3. AI 自動偵測並載入對應的 Skill
  4. 按照 Skill 定義的規則產生結構化回覆

💬 互動範例

> 請用 email-reply-helper 幫我回覆這封客戶詢價信：
您好，我們公司想詢問企業培訓方案的報價...

Claude: 您好，感謝您的詢問！
關於企業培訓方案，以下是我們的報價資訊...
（依照 SKILL.md 規則產生專業回覆）
🎉 跨平台成功！
同一個 SKILL.md，在 Antigravity 和 Claude Code 都能正常執行！不需要做任何修改。

📟 Claude Code CLI 常用指令

以下是 Claude Code 在終端機中最常用的指令，熟悉這些就能流暢操作。

指令 說明 範例
claude 進入互動模式 直接輸入 claude
claude -p "..." 單次執行模式 claude -p "幫我回覆這封信"
/help 查看內建指令列表 在互動模式中輸入
/skills 列出可用的 Skills 查看目前載入哪些 Skill
claude --version 查看版本 確認安裝成功
claude config 設定管理 管理 API Key 和偏好
💡 單次執行模式的好處
claude -p "..." 不需進入互動模式，一行指令直接執行，可以整合到腳本或自動化流程中！

🔄 一鍵同步到所有平台

為了避免在多個平台之間手動複製 Skill 檔案，我們用 skills-source/ 作為單一來源，搭配同步腳本一鍵發佈到所有平台。

同步流程

skills-source/（寫在這裡）
    ↓ bash sync-skills.sh
    ├── .agent/skills/　　（Antigravity 用）
    └── .claude/skills/　　（Claude Code 用）

同步腳本內容

#!/bin/bash
# sync-skills.sh - 一鍵同步 Skill 到所有平台

SKILL_SOURCE="skills-source"           # 來源資料夾
ANTIGRAVITY_DIR=".agent/skills"        # Antigravity 路徑
CLAUDE_DIR=".claude/skills"            # Claude Code 路徑

# 建立目標資料夾
mkdir -p "$ANTIGRAVITY_DIR"
mkdir -p "$CLAUDE_DIR"

# 同步檔案
cp -r "$SKILL_SOURCE"/* "$ANTIGRAVITY_DIR"/
cp -r "$SKILL_SOURCE"/* "$CLAUDE_DIR"/

echo "🎉 全部同步完成！"
✅ 一處維護，處處更新！
只要維護 skills-source/ 一個地方，所有平台都會同步更新！再也不用擔心版本不一致的問題。
