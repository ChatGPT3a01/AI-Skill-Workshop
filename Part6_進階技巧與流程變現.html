🛡️ 為什麼需要防呆機制？

「沒有防呆的 Skill，就像沒有護欄的山路——很危險！」

🚫

過度承諾

AI 可能承諾做不到的事，如未確認的交期、未授權的折扣

🔓

洩露機密

AI 可能不小心說出公司內部成本、競爭策略等敏感資訊

😡

語氣失當

AI 可能用不恰當的語氣回覆，讓客戶不滿

⚠️ 重要提醒：在正式使用 Skill 前，必須加入完整的防呆機制！

🔒 四大防呆指令實戰解析

🟢 MUST（必須遵守）

  • ✓ MUST 以「○○先生/小姐 您好」開頭
  • ✓ MUST 提供具體的下一步行動
  • ✓ MUST 以「敬祝 商祺」結尾

🔴 NEVER（絕對禁止）

  • ✗ NEVER 使用口語化表達（嗨、OK、沒問題啦）
  • ✗ NEVER 承諾未經確認的價格或交期
  • ✗ NEVER 洩露公司內部資訊
  • ✗ NEVER 提及競爭對手的負面評價

🟡 Check if（條件檢查）

  • 🔍 Check if 金額超過 10 萬 ➔ 建議加入主管副本
  • 🔍 Check if 金額超過 50 萬 ➔ Ask user 是否需要主管審核
  • 🔍 Check if 來信涉及法律爭議 ➔ 建議轉交法務部門

🔵 Ask user（詢問使用者）

  • 💬 Ask user 提供收件人的稱謂
  • 💬 Ask user 確認公司退貨政策
  • 💬 Ask user 提供確認的交期

📂 完整的 Skill 結構

展示 email-reply-helper 的完整檔案結構：

email-reply-helper/
├── SKILL.md              ← 主要指令檔（含防呆規則）
└── assets/
    ├── email-template.md   ← Email 格式範本
    ├── input-form.html     ← AI 生成器表單
    └── monetize-framework.html ← 流程變現圖
📄 SKILL.md
核心指令檔，定義 AI 的角色、流程、防呆規則。所有平台共用這一個檔案。
📄 email-template.md
Email 的格式模板，確保每次產出的 Email 格式一致、專業。
📄 input-form.html
AI 生成器的互動表單，讓不懂 AI 的人也能填空產生提示詞。
📄 monetize-framework.html
流程變現五層架構的視覺化展示圖。

🧪 防呆實測：在 Claude Code 測試

🔬 測試情境：故意輸入涉及法律爭議的客戶來信，看 AI 是否遵守 NEVER 規則

預期結果

AI 不會直接回覆法律問題，而是建議：
「本信件涉及法律事項，建議轉交法務部門處理」

✅ 測試通過！Claude Code 完全遵守了 SKILL.md 中的 NEVER 規則！

🧪 防呆實測：在 Gemini CLI 測試

同一個測試情境，在 Gemini CLI 上驗證——不同的 AI 平台都能遵守同一套防呆規則！

項目 Claude Code Gemini CLI
遵守 NEVER 規則 ✅ 是 ✅ 是
建議轉交法務 ✅ 有 ✅ 有
保持專業語氣 ✅ 是 ✅ 是
🎉 寫一次防呆規則，所有平台都有效！

🖱️ 打造你的專屬 AI 生成器

「如果你想讓不懂 AI 的同事也能用你的 Skill，怎麼辦？」

答案：做一個 HTML 表單（生成器），填空就能產生 AI 提示詞！

1
📝

填寫表單

客戶來信、優先順序、語氣

2
🚀

按下「產生 AI 提示詞」

一鍵生成格式化提示詞

3
📋

複製文字

貼到任何 AI 平台使用

⚠️ 重要：生成器本身不呼叫任何 AI API！它只是產生格式化的提示詞文字，使用者再手動複製到 AI 平台貼上使用。

⚡ 生成器產出的提示詞

展示操作流程：填好表單後，按下按鈕即可得到格式化的提示詞

📋 產出範例

## 請依照以下規則回覆 Email

角色：你是一位專業的客服人員
優先順序：高（24 小時內回覆）
語氣：正式且同理心

## 客戶來信內容
我在貴公司購買的產品出現瑕疵...

## 防呆規則
MUST 以「○○先生/小姐 您好」開頭
NEVER 承諾未經確認的價格或交期
MUST 以「敬祝 商祺」結尾

產出的文字可以複製到：

Claude Code Gemini CLI ChatGPT 任何 AI
