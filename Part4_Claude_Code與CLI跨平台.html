跨平台 Skill 部署與管理

在 Claude Code 和 Gemini CLI 上使用同一套 Skill

💻 Claude Code 🌟 Gemini CLI 🔑 API Key 🌐 跨平台

💻 認識 Claude Code

Claude Code 是 Anthropic 官方推出的 CLI 工具，讓你在終端機中直接與 Claude 互動。功能強大，但需要付費使用 API。

Claude Code 基本資訊

項目 說明
開發商 Anthropic（Claude 的開發公司）
費用 付費（按 API 用量計費）
安裝 npm install -g @anthropic-ai/claude-code
Skill 路徑 .claude/skills/
特色 強大的程式碼能力、終端機整合

📝 Step 1：註冊 Anthropic Console

要使用 Claude Code，首先需要到 Anthropic Console 註冊帳號並取得 API Key。

註冊步驟

  1. 前往 console.anthropic.com
  2. 點選 Sign Up（註冊）
  3. 填寫 Email 和密碼，完成註冊流程
💡 小提醒
免費註冊，但使用 API 需要先儲值（最低 $5 美元）。

🔑 Step 2：建立 API Key

登入 Anthropic Console 後，需要建立一組 API Key 來讓 Claude Code 驗證你的身份。

建立步驟

  1. 登入 Console 後台
  2. 左側選單點選 API Keys
  3. 點選 Create Key（建立金鍵）
  4. 輸入一個容易辨識的名稱
💡 命名建議
建議取一個容易辨識的名稱，如「my-skill-workshop」，方便日後管理多組金鍵。

📋 Step 3：複製 API Key

建立完成後，系統會顯示你的 API Key。它的格式是 sk-ant- 開頭的一長串字串。

⚠️ 極重要警告！
API Key 只會顯示一次！請立即複製保存到安全的地方。如果遺失，只能重新建立一個新的。千萬不要把 API Key 公開分享或上傳到 GitHub。

API Key 格式範例

sk-ant-api03-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx...
💡 保存建議
可以存在密碼管理器（如 1Password、Bitwarden）或本地的安全筆記中。

⚙️ Step 4：安裝 Claude Code

使用 npm 全域安裝 Claude Code，只要一行指令就能搞定。

安裝指令

npm install -g @anthropic-ai/claude-code

驗證安裝

claude --version
💡 前置需求
需要先安裝 Node.js（建議 v18 以上版本）。如果還沒安裝，可到 nodejs.org 下載安裝。

🔐 Step 5：使用 API Key 登入

安裝完成後，進入你的專案目錄，啟動 Claude Code 並使用 API Key 登入。

登入步驟

  1. 在專案目錄中執行 claude
  2. 選擇「Use an API key」
  3. 貼上你的 sk-ant-... API Key
  4. 出現提示符號 > 表示登入成功
✅ 登入成功！
看到 > 提示符號就表示可以開始使用了！

📂 專案層級 vs 個人層級 Skill

Claude Code 支援兩種 Skill 存放方式，依照使用範圍不同，選擇最適合的層級。

比較項目 專案層級 個人層級
路徑 .claude/skills/ ~/.claude/skills/
範圍 只有這個專案能用 所有專案都能用
比喻 工地的專用工具箱 你的隨身工具包
適合 團隊共用的 Skill 個人常用的 Skill
Git 會被版本控制 不會被版本控制
💡 課程建議
今天的課程使用「專案層級」，方便團隊分享和版本控制。

🚀 在 Claude Code 使用 Skill

把 SKILL.md 放到 .claude/skills/email-reply-helper/ 後，Claude Code 就會自動偵測並載入你的 Skill。

互動流程

  1. 將 SKILL.md 複製到 .claude/skills/email-reply-helper/
  2. 啟動 Claude Code，輸入客戶來信內容
  3. AI 自動偵測並載入對應的 Skill
  4. 按照 Skill 定義的規則產生結構化回覆
🎉 跨平台成功！
同一個 SKILL.md，在 Antigravity 和 Claude Code 都能正常執行！不需要做任何修改。

📟 Claude Code CLI 常用指令

以下是 Claude Code 在終端機中最常用的指令，熟悉這些就能流暢操作。

指令 說明 範例
claude 進入互動模式 直接輸入 claude
claude -p "..." 單次執行模式 claude -p "幫我回覆這封信"
claude --help 查看說明 列出所有可用選項
claude --version 查看版本 確認安裝成功

⚡ CLI 單次執行模式

使用 -p 參數可以不進入互動模式，直接執行一次性任務，非常適合整合到自動化流程中。

指令範例

claude -p "請用 email-reply-helper 幫我回覆這封客戶詢價信：
您好，我們公司想詢問企業培訓方案的報價..."

三大好處

🎯 不需進入互動模式
一行指令直接執行
🔗 可以整合到腳本中
串接自動化工作流
📦 適合批次處理
一次處理多封信件

🌟 認識 Gemini CLI

Gemini CLI 是 Google 推出的免費 CLI 工具，最大特色是自動讀取工作區檔案，能直接辨識目錄中的 Skill 定義。

安裝方式

npm install -g @google/gemini-cli

（或依照 Google 官方文件指引安裝）

三種認證方式

方式 說明 推薦
Login with Google 用 Google 帳號登入（免費） ⭐ 推薦
API Key 使用 Gemini API Key 進階用
Vertex AI GCP 企業版 企業用
💡 最簡單的方式
選「Login with Google」最簡單，免費就能用！有 Google 帳號就能直接開始。

🔄 一鍵同步到所有平台

為了避免在多個平台之間手動複製 Skill 檔案，我們用 skills-source/ 作為單一來源，搭配同步腳本一鍵發佈到所有平台。

同步流程

skills-source/（寫在這裡）
    ↓ bash sync-skills.sh
    ├── .agent/skills/　　（Antigravity 用）
    └── .claude/skills/　　（Claude Code 用）

同步腳本內容

#!/bin/bash
# sync-skills.sh - 一鍵同步 Skill 到所有平台

SKILL_SOURCE="skills-source"
ANTIGRAVITY_DIR=".agent/skills"
CLAUDE_DIR=".claude/skills"

# 建立目標資料夾
mkdir -p "$ANTIGRAVITY_DIR"
mkdir -p "$CLAUDE_DIR"

# 同步檔案
cp -r "$SKILL_SOURCE"/* "$ANTIGRAVITY_DIR"/
cp -r "$SKILL_SOURCE"/* "$CLAUDE_DIR"/

echo "🎉 全部同步完成！"
✅ 一處維護，處處更新！
只要維護 skills-source/ 一個地方，所有平台都會同步更新！再也不用擔心版本不一致的問題。
