Part 3 · 在 Antigravity 執行 Skill

讓你的 Skill 在 Antigravity 上跑起來

從登入平台到成功執行，完整實作流程

🚀 Antigravity 🆓 免費平台 ⚡ 即時測試

🌐 認識 Antigravity 平台

Antigravity 是免費的 AI 開發環境，不需要 API Key，註冊即可使用。它是今天課程最主要的實作平台，讓你在瀏覽器中就能完成所有操作。

平台特色

  • 免費使用（不需要 API Key）
  • 網頁版 IDE（不需要安裝任何東西）
  • 內建 AI 對話面板
  • 完整支援 Agent Skills
💡 提示：Antigravity 是今天最主要的實作平台，所有學員都可以免費使用！

🖥️ Antigravity IDE 介面導覽

進入 Antigravity 後，你會看到一個完整的 IDE 環境，主要分為三大區域：

📂

左側面板

檔案管理器，可以新增、刪除、重新命名檔案和資料夾

📝

中央區域

程式碼 / 文字編輯器，直接編輯 SKILL.md

💬

右側面板

AI 對話面板，與 AI 互動並測試 Skill

💬 AI 對話面板操作

AI 對話面板是你與 Skill 互動的主要介面。操作流程非常直覺：

  1. 在右側面板輸入你的需求
  2. AI 會自動偵測可用的 Skills
  3. 如果你的需求符合某個 Skill 的 description，AI 會自動載入該 Skill
  4. AI 按照 SKILL.md 的規則執行
💡 重要觀念：你不需要手動告訴 AI「使用 email-reply-helper」，AI 會根據 description 自動判斷！

📁 在 Antigravity 建立 Skill 目錄

Antigravity 的 Skill 路徑為 .agent/skills/，所有 Skill 都必須放在這個目錄下：

目錄結構

.agent/skills/
└── email-reply-helper/
    ├── SKILL.md
    └── assets/
        └── email-template.md
⚠️ 注意：路徑必須是 .agent/skills/（注意前面的點），放錯位置 AI 就找不到！

✅ 確認 Skill 結構正確

在開始測試之前，請逐一確認以下項目：

  • ☐ .agent/skills/email-reply-helper/ 資料夾存在
  • ☐ SKILL.md 放在正確位置
  • ☐ assets/ 資料夾已建立
  • ☐ email-template.md 已放入 assets/
✅ 結構正確！現在可以開始測試了

🔨 第一次測試：詢價回覆

現在來測試我們的 Skill 是否能正確運作。在 AI 對話面板中輸入以下內容：

📩 測試輸入：
「您好，我們公司想詢問貴司 A 型號產品 500 組的報價，希望能在下個月 15 日前交貨，請問可以嗎？」

這是一封典型的詢價信，AI 應該要：

  1. 判斷來信類型為「詢價」
  2. 分析出三個需求：報價、數量、交期
  3. 按照 SKILL.md 的格式回覆

✨ AI 按照 Skill 規範回覆

以下是 AI 按照 SKILL.md 規則產出的回覆結構：

📊

分析摘要

情緒判斷 + 回覆類型 + 重點需求，讓你快速掌握來信重點

📧

回覆信件

標準格式，以「○○先生/小姐 您好」開頭，「敬祝 商祺」結尾

📋

執行摘要

來信分類 + 處理方式 + 風險提醒 + 建議後續

✅ 成功！AI 完全按照 SKILL.md 的規則執行！這就是 Skill 的威力

😤 第二次測試：客訴情境

同一個 Skill，面對不同類型的來信，AI 會自動調整回覆策略。來看看客訴情境的表現：

詢價 vs 客訴 回覆比較

項目詢價回覆客訴回覆
情緒判斷中性負面
回覆類型詢價客訴
語氣調整正式友善同理心 + 解決方案
⚠️ 重要提醒：測試不同情境是確保 Skill 品質的關鍵步驟！

⚠️ 常見 YAML 錯誤與除錯

當 Skill 不如預期運作時，先檢查以下常見的 YAML 格式問題：

錯誤類型問題描述正確寫法
縮排不一致 混用 Tab 和空格 統一使用 2 個空格
忘記 > 符號 description 後面沒加 > description: >
name 不一致 name 和資料夾名稱不同 兩者必須完全一致
特殊字元 冒號、引號等未正確處理 用引號包起來
缺少空格 name:value name: value（冒號後要空格）
💡 除錯心法：遇到 Skill 不生效時，90% 是 YAML 格式問題！先檢查這些常見錯誤
