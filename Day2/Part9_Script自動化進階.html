😫 手動處理的痛苦

想像一下：你剛開完會，手上有一個 30 分鐘的會議錄音檔。

你需要：把它轉成逐字稿 → 整理成會議紀錄 → 轉成漂亮的 Word 文件交給主管

手動處理的流程

步驟手動做法花費時間
語音轉文字找工具上傳、慢慢等轉錄，或邊聽邊打字30-60 分鐘
整理紀錄複製文字、手動調整格式、分類議題20-30 分鐘
轉 Word開 Word、排版、貼標題、調格式15-20 分鐘
合計1-2 小時
💡 但如果用 Claude Skills 串聯，只要下 三個簡單指令 就能搞定！

🧩 Skill 的三大元素

Day 1 和 Day 2 上午我們學了前兩個，現在加上第三個：Scripts

1️⃣ SKILL.md

給 AI 看的說明書

定義觸發條件、執行規則、輸入輸出格式、MUST/NEVER 規範

2️⃣ 範本與參考資料

assets/ 資料夾

範本檔案、參考格式、風格指引，讓 AI 有樣學樣

3️⃣ Scripts 執行腳本

真正在「幹活」的程式

Python 腳本，可以呼叫外部 API、處理檔案、自動化操作

🔑 有了 Scripts，你的 Skill 就像裝了引擎——可以呼叫外部 API、執行複雜的資料處理、自動操作檔案、整合第三方工具。

📂 Audio-to-Text Skill 完整結構

audio-to-text/
├── SKILL.md                      ← 給 AI 看的說明書
│   ├── 觸發條件：何時使用此 Skill
│   ├── 執行流程：從 Input 讀取音檔
│   ├── 呼叫 OpenAI Whisper API 轉繁中逐字稿
│   └── 輸出到 Output 資料夾
│
├── README.md                     ← 給人類看的快速指南
│
└── scripts/
    └── audio_transcriber.py      ← 真正執行的 Python 程式
        ├── 呼叫 Whisper API 語音辨識
        ├── 利用大語言模型翻譯修正
        ├── 根據語意自動斷句
        ├── 保留專業術語
        ├── 處理超大檔案自動分割
        └── 批次處理多個音檔
✅ 由 SKILL.md 指揮，audio_transcriber.py 負責執行，兩者完美配合！

📥 如何下載安裝 Skill？（手把手教學）

以安裝官方 docx Skill 為例，完整示範：

方式一：手動下載（最通用、最好懂）

Terminal
# Step 1：下載官方 Skills 倉庫
$ git clone https://github.com/anthropics/skills.git
# Step 2：把你要的 Skill 複製到 Claude Code 目錄
$ cp -r skills/docx ~/.claude/skills/
$ cp -r skills/pdf ~/.claude/skills/
$ cp -r skills/pptx ~/.claude/skills/
# Step 3：確認安裝成功
$ ls ~/.claude/skills/
docx/ pdf/ pptx/
✅ 安裝完成！下次開啟 Claude Code 就能自動使用

方式二：在 Claude Code 中直接安裝

Claude Code
claude> 幫我從 github.com/anthropics/skills 安裝 docx 和 pdf skill
好的，我來幫你下載並安裝...
- git clone anthropics/skills... ✅
- 複製 docx/ 到 ~/.claude/skills/... ✅
- 複製 pdf/ 到 ~/.claude/skills/... ✅
✅ 安裝完成！你現在可以使用 docx 和 pdf skill 了。
⚠️ 專案層級 vs 全域：放在 .claude/skills/（專案目錄下）只對該專案生效；放在 ~/.claude/skills/（家目錄）則所有專案都能用。

🔄 Skill 安裝後如何運作？

Skill 安裝後，Claude 會自動偵測並在需要時載入：

階段 1：掃描
🔍
Claude 啟動時掃描 skills/ 目錄
只讀取每個 SKILL.md 的 metadata（約 100 tokens）
知道「有哪些 Skill 可用」
階段 2：匹配
🎯
當你下指令時，Claude 判斷
「這個任務適合用哪個 Skill？」
載入完整指令（< 5,000 tokens）
階段 3：執行
⚡
按照 SKILL.md 的規則執行
如果有 scripts/，會執行 Python 腳本
產出結果到指定位置
💡 你不需要手動「啟動」任何 Skill — 只要把 Skill 資料夾放對位置，Claude 就會在需要的時候自動使用它。就像手機 App 裝好後，需要時打開就能用。

⚠️ 兩個重要提醒

🔑 提醒一：API Key 安全

記得在 .env 設定你的 OpenAI API Key

千萬不要把 .env 傳到 GitHub！

記得加進 .gitignore

💰 提醒二：成本控管

因為是呼叫付費 API（Whisper）

記得留意你的 API 使用量與計費方式

建議先用短音檔測試
