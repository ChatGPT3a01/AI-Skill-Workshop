📚 什麼是 Google NotebookLM？

NotebookLM 是 Google 推出的 AI 筆記本工具，基於 RAG（檢索增強生成）架構，能將你的文件轉化為 AI 可查詢、可生成內容的智慧知識庫。

NotebookLM 能做什麼？

📎上傳各種來源

PDF、網頁 URL、YouTube 影片、Google Drive 文件、直接貼上文字，全部丟進去就能分析

🎙️自動生成內容

一鍵生成 Podcast、影片、簡報、測驗題、閃卡、報告、心智圖、資訊圖表

🔍智慧問答

對你上傳的文件提問，AI 會引用原文回答，不會亂掰——因為是 RAG 架構，回答都有根據

💡 重要：NotebookLM 的 AI 回答「只根據你上傳的文件」，不會混入外部資料。這就是 RAG 架構的優勢——每句回答都可追溯到原始來源。

😫 手動操作 NotebookLM 的痛點

NotebookLM 功能很強，但全部都要手動點、手動選。如果你有批次需求，這些問題馬上浮現：

痛點具體情境浪費時間
逐一上傳要分析 20 篇 PDF 論文，得一個一個拖拉上傳30+ 分鐘
手動建立每門課程都要手動建立新筆記本、命名、設定15+ 分鐘
點選生成每個筆記本都要手動點「生成 Podcast」、等待、下載20+ 分鐘
無法匯出閃卡、心智圖只能線上看，無法下載為可編輯格式無解
沒有 APIGoogle 至今未開放 NotebookLM 的官方 API無法程式化
💡 核心問題：Google 沒有提供 NotebookLM API！所有操作都必須透過網頁介面手動完成。如果要處理大量資料，效率極低。
⬇️
✅ 解決方案：透過 notebooklm-py 逆向工程 + Flask GUI + NLP 解析器，讓你用「自然語言」就能控制 NotebookLM！

🔧 notebooklm-py 是什麼？

notebooklm-py 是一個開源的 Python CLI 工具，透過逆向工程 NotebookLM 的未公開 API，讓你能用程式碼操控 Google NotebookLM 的所有功能。

🔒 github.com/siddicky/notebooklm-py
siddicky / notebooklm-py Public

Python client for Google NotebookLM — Automate notebooks, sources, and audio generation

⭐ 1.2k stars 🍴 89 forks 📜 MIT License 🐍 Python

📖 README.md

notebooklm-py — A Python client for Google NotebookLM

pip install notebooklm-py

核心特色

  • 逆向工程：透過 Playwright 瀏覽器自動化，模擬真實使用者操作
  • 完整功能：涵蓋 NotebookLM 網頁版的所有功能，甚至更多（可匯出閃卡、心智圖等）
  • 三種使用方式：CLI 指令、Python API、Claude Code Skills
  • Cookie 認證：不需要儲存帳號密碼，用瀏覽器登入一次後自動保存
  • JSON 輸出：所有操作結果都以 JSON 格式回傳，方便程式處理
  • 開源免費：完全開源，可自由使用與修改

🧠 NLP Parser：自然語言如何變成操作？

這是整個系統最核心的部分——NLP Parser（自然語言解析器）負責把你說的話，轉換成系統可執行的指令。

解析流程

使用者輸入
「幫我生成 Podcast」
⬇️
Step 1：NLP Parser 解析
關鍵字匹配：找到「生成」+「Podcast」
→ 識別意圖：generate_audio　→ 信心度：0.9
⬇️
Step 2：Execute API 接收意圖
intent = "generate_audio"
→ 呼叫 notebooklm_service.generate_audio()
⬇️
Step 3：notebooklm-py 執行 CLI 指令
$ notebooklm generate audio --json
⬇️
Step 4：Playwright 瀏覽器自動化
自動開啟 Chromium → 點擊「生成」按鈕 → 等待完成
⬇️
Step 5：結果回傳到前端 ✅
{ "success": true, "data": {...} }

三種 NLP 解析模式比較

模式原理準確度費用適用場景
關鍵字匹配比對預設關鍵字清單中等免費入門使用、確定指令
Gemini APIGoogle AI 理解語意高需 API Key模糊指令、複雜需求
OpenAI APIGPT 理解語意高需 API Key模糊指令、複雜需求
💡 建議：先用「關鍵字匹配」模式（免費），熟悉系統後再考慮升級到 Gemini 或 OpenAI 模式。關鍵字匹配在指令明確時就夠用了！

🏷️ NLP 意圖分類：五大操作類別

NLP Parser 會把你的自然語言指令歸類到以下五大意圖類別，每個類別對應不同的 notebooklm-py 操作：

意圖類別對應關鍵字觸發的 CLI 指令
📚 筆記本管理列出、建立、刪除、重命名筆記本notebooklm list / create / delete
📎 來源管理新增網址、加入 YouTube、匯入 PDFnotebooklm source add
🎙️ 內容生成生成 Podcast、影片、簡報、測驗、閃卡、報告、心智圖notebooklm generate audio/video/...
🔍 查詢問答問、查詢、搜尋、問答notebooklm chat
📥 下載匯出下載、匯出、儲存notebooklm download

關鍵字匹配的程式碼原理

INTENT_PATTERNS = {
    "list_notebooks": [
        "列出筆記本", "顯示筆記本", "我的筆記本", "有哪些筆記本"

    ],
    "generate_audio": [
        "生成Podcast", "製作音訊", "產生播客", "生成播客"
    ],
    "add_source_url": [
        "新增網址", "加入網頁", "匯入URL"
    ],
    # ... 更多意圖模式
}

  • 語意理解更精準
  • 能處理模糊、口語化的指令

  • 需要 OpenAI API Key
  • 同樣高精度語意理解
  • 適合已有 OpenAI Key 的使用者

  • 開放 5000 port 時注意防火牆設定

❓ 常見問題與疑難排解

使用過程中如果遇到問題，可以參考以下解決方案：

問題可能原因解決方案
「No notebook specified」錯誤 未選擇筆記本就執行需要筆記本的操作 先在左側下拉選單選擇筆記本
登入狀態失效 Cookie 過期或被清除 重新執行 notebooklm login
Podcast 很久沒反應 生成需要 2-5 分鐘 檢查右側「任務狀態」欄位，任務在背景執行中
NLP 解析不準確 關鍵字匹配模式有限制 使用更明確的關鍵字，或切換到 Gemini/OpenAI 模式
無法遠端存取 防火牆阻擋或 host 設定問題 確認 host='0.0.0.0' 並開放 5000 port
pip install 失敗 Python 版本過舊或 pip 版本問題 確認 Python 3.9+ 並執行 pip install --upgrade pip
Playwright 安裝失敗 網路問題或權限不足 使用管理員身分執行，確認網路暢通
🚨 風險提醒：notebooklm-py 是透過逆向工程建立的工具，Google 可能隨時修改 NotebookLM 介面導致工具暫時失效。建議：
  • 使用專用 Google 帳號測試，避免影響主帳號
  • 定期更新：pip install --upgrade notebooklm-py
  • 不適合用於需要 100% 穩定性的正式生產環境
