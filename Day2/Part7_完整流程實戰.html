J-2｜各場景快速起步模板（A）

---
name: lesson-plan-creator
description: >
  當使用者提供教學主題、年級與時數時觸發。
  自動產生符合 108 課綱的完整教案，
  包含學習目標、教學流程與評量規準。
---

## 前置檢查
- Check if 主題明確（非過於寬泛，如「數學」）
- Check if 年級已指定（國小/國中/高中 + 幾年級）
- Check if 時數合理（1–4 節課）
- Ask user 核心素養指標（如未提供）

## Step 1：設定學習目標
- MUST 符合 108 課綱核心素養
- MUST 條列 3–5 個具體可評量的學習目標
- MUST 使用「能…」開頭的行為動詞

## Step 2：設計教學流程
- MUST 包含引起動機、發展活動、綜合活動三段
- MUST 每節課至少 2 個師生互動設計
- NEVER 讓單一講述超過 15 分鐘

## Step 3：設計評量
- MUST 含形成性評量與總結性評量
- NEVER 只使用紙筆測驗
- MUST 提供評量規準（rubric）

## 輸出格式
以 Markdown 表格呈現完整教案，含時間分配欄位

J-2｜快速起步模板（D / F）

場景 D：客服分類器

---
name: ticket-classifier

  當使用者貼上客服工單或客戶訊息時觸發。

## 前置檢查
- Check if 訊息包含可識別的問題描述

## Step 1：判斷問題類型
退款 / 技術問題 / 帳號問題 / 一般詢問 / 投訴

## Step 2：標注優先級

## 安全規則
- NEVER 承諾具體賠償金額
- NEVER 透露內部 SLA 數字

## 輸出格式
分類：[類型] | 優先級：[P0/P1/P2] | 摘要：一句話

J-2｜常見卡關點與解法

卡關狀況解法
不知道從哪裡開頭先寫 description，描述「什麼情況觸發」
步驟太籠統想像新人看到這個步驟，能不能直接執行？
不確定輸出格式先手動做一次，把結果的格式複製下來
Skill 越寫越大超過 50 行就拆成兩個 Skill
YAML 報錯檢查縮排（用空格不用 Tab），冒號後面要有空格
AI 不觸發 Skilldescription 要包含明確的觸發情境關鍵字
