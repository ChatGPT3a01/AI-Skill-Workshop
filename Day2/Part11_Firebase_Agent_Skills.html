核心概念

📦模組化知識包

每個 Skill 是一個「知識模組」，包含 Firebase 特定服務的專業指令、最佳實踐、自動化腳本，讓 AI 不用讀整份文件就能正確操作

💰省 Token 設計

採用漸進式揭露（Progressive Disclosure）架構——AI 只載入當下需要的知識，不會一口氣灌入整份文件，大幅降低 Token 消耗

🌐跨平台支援

不只 Gemini CLI！同時支援 Antigravity、Claude Code、Cursor、GitHub Copilot 等 30+ 種 AI 助理，一套 Skill 到處用

💡 白話解釋：就像我們課堂上自己寫的 SKILL.md 一樣，但這是 Google 官方幫你寫好的——專門針對 Firebase 各項服務的「超詳細說明書」，裝上去後 AI 就變成 Firebase 專家！

🧩 Firebase 的 AI 輔助開發三大工具

Google 為 Firebase 開發者提供了三個互補的 AI 工具，各司其職：

📚Agent Skills（知識）

「知道怎麼做」

提供 Firebase 各服務的專業知識、最佳實踐與流程指引。

好比一個資深 Firebase 顧問在旁邊指導你

🔌MCP Server（能力）

「有能力執行」

讓 AI 能直接讀寫 Firebase 專案資源（資料庫、設定、使用者等）。

好比給 AI 一把直接操作 Firebase 的鑰匙

⌨️Firebase CLI（工具）

「實際操作的手」

命令列工具，執行部署、初始化、管理等實際操作。

好比真正動手施工的工人

三者的協作關係： 你：「幫我建一個有登入功能的 Todo App，部署到 Firebase」 ⬇️ Agent Skills → 告訴 AI：要用 Firebase Auth + Firestore + App Hosting ⬇️ MCP Server → 讓 AI 讀取你的 Firebase 專案設定、建立資料庫規則 ⬇️ Firebase CLI → 實際執行 firebase deploy 部署到雲端

🧠 核心技術：漸進式揭露（Progressive Disclosure）

Firebase Agent Skills 最精妙的設計是漸進式揭露——AI 不會一次載入所有知識，而是「用到什麼才載入什麼」。

傳統做法 vs 漸進式揭露

❌ 傳統做法：全部灌入

把整份 Firebase 文件（10 萬字）
丟進 AI 的 context

問題：
• 消耗大量 Token（$$$$）
• AI 容易搞混資訊
• context 空間被佔滿
• 回應速度變慢

✅ 漸進式揭露：按需載入

只載入 SKILL.md 的 description（幾十字）
AI 需要時才載入對應章節

優勢：
• Token 消耗大幅降低
• AI 聚焦在當下任務
• context 空間留給你的程式碼
• 回應更快更準確

漸進式揭露的運作流程： 你：「幫我加 Google 登入功能」 ⬇️ AI 讀取所有 Skill 的 description（各幾十字） ⬇️ 匹配到 firebase-auth-basics（「身份認證」） ⬇️ 載入 firebase-auth-basics 的完整內容 ⬇️ 只有這一個 Skill 被載入！其他五個不會佔用空間
💡 設計啟示：這跟我們自己寫 Skill 時的 description 很重要是同一個道理——description 越精準，AI 越能在對的時機選到對的 Skill！

🧹 清理資源與注意事項

實作完成後，記得清理資源以避免不必要的費用：

清理方式一：刪除 Cloud 專案（最徹底）

$ gcloud projects delete $GOOGLE_CLOUD_PROJECT

這會刪除整個 Cloud 專案及其所有資源，包括 Firebase、Firestore、部署的 App。

清理方式二：只刪除自訂 Skill

$ rm -rf ~/.gemini/skills/my-favorite-things

這只會刪除我們練習用的 my-favorite-things Skill，不影響 Firebase Skills。

重要注意事項

項目說明
費用實作預估花費不到 $1 美元。刪除 Cloud 專案後不會再產生費用
Firebase 免費方案Spark（免費）方案有 Firestore 每日讀寫限額，足夠練習使用
Agent Skills 保留Firebase Agent Skills 裝在本機，不需要清理，下次直接用
登入狀態firebase logout 可以登出 Firebase CLI
🚨 絕對不要執行：sudo rm -rf $HOME——這會刪除你整個使用者目錄，無法復原！Codelab 中有提到這個警告。
