🔐 Step 2：加上登入 → firebase-auth-basics

gemini> 加上 Google 登入功能。
       登入後根據 email 網域區分角色：
       @gm.school.edu.tw 結尾是學生，其他是教師。

AI 產出的前端登入程式碼

import { getAuth, signInWithPopup, GoogleAuthProvider,
         onAuthStateChanged } from "firebase/auth";

const auth = getAuth();

async function login() {
  const provider = new GoogleAuthProvider();
  const result = await signInWithPopup(auth, provider);
  const email = result.user.email;
  // 根據 email 判斷角色
  const role = email.endsWith("@gm.school.edu.tw")
    ? "student" : "teacher";
  console.log(`歡迎，${result.user.displayName}（${role}）`);
}

// 監聽登入狀態
onAuthStateChanged(auth, (user) => {
  if (user) {
    const role = user.email.endsWith("@gm.school.edu.tw")
      ? "student" : "teacher";
    showDashboard(role);
  } else {
    showLoginPage();
  }
});
💡 一句話就搞定：Google 登入 + 角色判斷 + 狀態監聽，全部由 AI 讀取 firebase-auth-basics 後自動產出！

🤖 Step 4：加 AI 助教 → firebase-ai-logic-basics

gemini> 加入 AI 助教功能：
   1. 學生：AI 檢查反思寫得好不好
   2. 教師：AI 產生回饋建議草稿
   3. 學期末：自動產生 200 字學習歷程摘要

三個 AI 功能

// 引入 Firebase AI 模組
import { getAI, getGenerativeModel, GoogleAIBackend } from "firebase/ai";

// 初始化 Gemini AI（使用 Google AI 後端）
const ai = getAI(app, { backend: new GoogleAIBackend() });
// 選擇模型：gemini-2.5-flash-lite（輕量快速版）
const model = getGenerativeModel(ai, { model: "gemini-2.5-flash-lite" });

// ========== 功能 1：學生的 AI 助教 ==========
// 學生貼上自己的反思，AI 檢查寫得好不好、給 3 點建議
async function aiCheckReflection(reflection) {
  const result = await model.generateContent(
    `你是高中自主學習指導老師。請檢查這段反思：「${reflection}」
     用鼓勵的語氣，給出 3 點具體建議。`
  );
  return result.response.text();
}

// ========== 功能 2：教師的 AI 輔助批閱 ==========
// AI 先讀學生週誌，自動產生 50 字以內的回饋建議草稿
async function aiGenerateFeedback(content, goal) {
  const result = await model.generateContent(
    `學習目標：${goal}  本週內容：${content}
     撰寫 50 字以內的教師回饋草稿。`
  );
  return result.response.text();
}

// ========== 功能 3：學期末自動摘要 ==========
// 把 18 週週誌摘要成 200 字學習歷程總結
async function aiGenerateSemesterSummary(allLogs) {
  const logsText = allLogs.map(
    log => `第${log.week}週：${log.content}`
  ).join("\n");
  const result = await model.generateContent(
    `將以下週誌摘要為 200 字學習歷程總結：\n${logsText}`
  );
  return result.response.text();
}

🧠 核心技術：漸進式揭露（Progressive Disclosure）

Firebase 完整文件超過 10 萬字。如果全部塞給 AI vs 用 Skill：

❌ 整份文件塞給 AI

• Token 消耗：10 萬+ tokens
• 準確度：資訊太多，容易搞混
• 回應速度：慢
• 成本：高

✅ 用 Skill（漸進式揭露）

• Token 消耗：幾百 tokens
• 準確度：聚焦當下任務，精準回答
• 回應速度：快
• 成本：低

運作原理

Step 1：AI 啟動時，只讀每個 Skill 的 description（每個幾十字） → 七個 Skill 總共才幾百字 Step 2：你說「幫我加 Google 登入」 → AI 比對 description，發現 firebase-auth-basics 最匹配 → 只載入這一個 Skill 的完整內容 → 其他六個 Skill 不佔空間 Step 3：你說「建 Firestore 資料庫」 → AI 再載入 firebase-firestore-basics → 之前用完的 auth Skill 已經不需要了
💡 這跟我們自己寫 Skill 時 description 要精準是同一個道理！description 寫得好，AI 才能正確配對。

→ 安裝七大 Skill：basics / auth / firestore / data-connect / ai-logic / hosting / app-hosting
